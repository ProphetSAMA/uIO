## Project Overview

uIO (校园电费查询平台) is a campus dormitory electricity fee inquiry platform. It queries electricity balances and consumption history from a third-party campus payment API (59wanmei.com), persists the data, and visualizes it through charts and tables.

The project is a full-stack application with a Spring Boot backend and a Vue 3 frontend.

## Tech Stack

- **Backend**: Spring Boot 3.3.3, Java 17, Maven, Spring Security, JWT (jjwt 0.11.2), MyBatis-Plus 3.5.7, JPA (Hibernate), Redis (Spring Cache), Apache HttpClient, JavaMail
- **Frontend**: Vue 3, Vite, TypeScript, Pinia, Vue Router 4, Element Plus, Axios, ECharts / VChart
- **Database**: MySQL

## Development Commands

### Backend

```bash
# Run the backend (requires MySQL and Redis running locally)
./mvnw spring-boot:run

# Build JAR (skip tests)
./mvnw clean package -DskipTests

# Run tests
./mvnw test
```

### Frontend

```bash
cd wsssfun-ui

# Install dependencies
npm install

# Start dev server
npm run dev

# Build for production
npm run build
```

### Database Setup

1. Create a MySQL database named `uio`.
2. Update `src/main/resources/application.yml` with your local MySQL/Redis credentials (the committed file contains hardcoded local credentials).
3. The application uses JPA `ddl-auto: update`, so schema is generated automatically. There are no SQL migration files in this repository.

## Architecture Notes

### Backend (`src/main/java/fun/wsss/uio/`)

- **Layered architecture**: `controller` -> `service` (interface + `impl`) -> `mapper` -> `model`.
- **MyBatis-Plus**: Mappers extend `BaseMapper`. The main application class has `@MapperScan("fun.wsss.uio.mapper")`.
- **Room hierarchy**: Users select a dorm room during registration. The hierarchy is `Building` -> `Floor` -> `Room`, modeled in `model/room/` and queried via cascading endpoints in `RoomController`.
- **Security**: `WebSecurityConfig` disables CSRF and permits all requests (`/**`). Passwords are encoded with `BCryptPasswordEncoder`. JWT tokens are issued on login and validated on protected endpoints.
- **Scheduled tasks**: `ScheduledTasks.java` runs two hourly cron jobs (`0 0 * * * ?`):
  1. `insertLastedPowerValue()` – fetches the latest electricity value from the external API and inserts it into the `power` table.
  2. `refreshAllPowerValue()` – refreshes the `allPowerValue` Redis cache.
- **External API integration**: `Http.java` hardcodes a POST request to `cloudpaygateway.59wanmei.com:8087` to fetch room electricity state. `Json.java` parses the response to extract the current `quantity` value. This is the core data source for the application.
- **Caching**: Spring Cache with Redis is used for `allPowerValue` and `recentWeekPowerValue`.

### Frontend (`wsssfun-ui/src/`)

- **Entry point**: `main.ts` mounts the Vue app, initializes Pinia and Vue Router, and sets up Element Plus with Chinese locale.
- **Routing**: `router/index.js` defines routes (`/`, `/login`, `/register`, `/info-power`). A global before-each guard checks `sessionStorage.getItem('token')` for routes with `meta.requiresAuth`.
- **State management**: Pinia stores are in `store/` (`user.js`, `powerStore.js`, `roomStore.js`).
- **API client**: `utils/axios.js` creates an Axios instance with `baseURL: '127.0.0.1/api'` and a request interceptor that attaches `Bearer ${token}` from `sessionStorage`.
- **Auto-imports**: Vite config uses `unplugin-auto-import` and `unplugin-vue-components` with `ElementPlusResolver`, so Element Plus components and APIs do not need explicit imports.
- **Styling**: Global styles are in `assets/styles/global.css`. Custom fonts and icon fonts are in `assets/fonts/` and `assets/iconfont/`.

### Build & CI

- The root `pom.xml` only builds the backend. The frontend is built separately with npm.
- `Jenkinsfile` builds both: it runs `mvn clean package -DskipTests` in the repo root and `npm run build` inside `wsssfun-ui`, then archives `target/*.jar` and `wsssfun-ui/dist/**`.

## Important Quirks

- **No linting or formatting tools** are configured (no ESLint, Prettier, Checkstyle, or Spotless).
- **Tests are minimal**: The only test class (`UIoApplicationTests.java`) is commented out.
- **Hardcoded external API**: The electricity data source (`Http.java`) has hardcoded headers, cookies, and a request body pointing to a specific room (`2-1--3-7301`). If the external API changes, this class must be updated.
- **Frontend API baseURL is literal**: `axios.js` uses `baseURL: '127.0.0.1/api'` without a protocol (`http://`). Depending on the browser context, this may resolve unexpectedly; verify the actual runtime base URL when debugging.
- **No SQL seed files**: Database schema is generated by Hibernate (`ddl-auto: update`). Initial room/building/floor data must be inserted manually or already exist in the database.
